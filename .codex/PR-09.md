# PR-09: Define bootstrap output contract types + JSON serialization

## Goal
Standardize outputs from platform_install/upgrade execution.

## Scope
Types only + wiring to print/store outputs.

## Changes
- Add `BootstrapOutput` type:
  ```json
  {
    "config_patch": { ... },
    "secrets_writes": [
      { "key": "...", "value": "...", "scope": "...", "metadata": { ... } }
    ],
    "warnings": [ ... ],
    "ready": true
  }
Ensure deployer:

captures this output from the bootstrap flow runner

prints it to stdout in --json mode

persists a copy under bootstrap state dir (non-secret parts only)

Acceptance Criteria

Output JSON is stable and versioned (include output_version if needed).

Non-secret parts can be stored without leaking secrets.

Tests

Serialize/deserialize tests for BootstrapOutput.

Ensure persistence excludes secret values if you store summaries.

Notes

Secret values must not be written to bootstrap state.
