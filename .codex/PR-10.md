# PR-10: Introduce installer host API: capability negotiation (host side)

## Goal
Implement the host-provided capability surface so installer can choose interaction mode.

## Scope
Host-side only; installer.wasm can be stubbed until later PR.

## Changes
- Define a host capability model:
  - available adapters: CLI, JSON (future HTTP/MQTT placeholders)
  - allowed transports/policy: “no listeners”, “offline only”, etc.
- Add deployer flags:
  - `--interaction cli|json|auto`
  - `--no-listeners`
  - `--offline-only`
- Make capabilities available to the bootstrap runtime (for later WIT wiring).

## Acceptance Criteria
- `platform install --interaction auto` produces a capability set.
- Policy flags remove or deny certain adapters.

## Tests
- Unit tests for capability construction under various flags.

## Notes
- No WIT required yet; just internal model.
