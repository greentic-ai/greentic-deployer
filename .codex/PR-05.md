# PR-05: Deployer: offline signature/digest verification hook (policy gate)

## Goal
Verify pack signatures/digests offline before any bootstrap action.

## Scope
Add verification step + policy options; keep defaults permissive unless CI/flag requires strict.

## Changes
- Add verification pipeline for platform commands:
  - compute/confirm digests
  - verify signatures/attestations if present
- Add flags:
  - `--verify` (default true if signatures exist; otherwise warn)
  - `--strict-verify` (fail if signatures missing/invalid)
- Log verification results deterministically.

## Acceptance Criteria
- Valid signatures pass; invalid signatures fail.
- Missing signatures:
  - `--verify` => warning + continue
  - `--strict-verify` => fail

## Tests
- Test: missing signatures -> warning/continue (non-strict)
- Test: missing signatures -> fail (strict)
- Test: invalid signature -> fail

## Notes
- Keep policy in deployer; do not push into installer.
