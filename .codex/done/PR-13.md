# PR-13: Implement JSON (non-interactive) adapter (automation-friendly)

## Goal
Support air-gapped automation and CI without interactive prompts.

## Scope
JSON input/output paths.

## Changes
- Add flags:
  - `--answers ./answers.json` (or `--answers @-` for stdin)
  - `--output ./bootstrap_output.json` (optional)
- JSON adapter feeds answers to installer and returns BootstrapOutput.

## Acceptance Criteria
- `platform install --interaction json --answers answers.json` succeeds.
- `--output` writes output JSON deterministically.

## Tests
- Integration test using fixture answers JSON.

## Notes
- Useful for k8s GitOps style bootstraps later.
