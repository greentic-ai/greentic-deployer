# PR-25: Kubernetes secrets backend (optional) via secrets-store routing

## Goal
Enable secrets writes to Kubernetes Secrets while preserving “installer emits intent, deployer executes”.

## Scope
Add a k8s secrets-store implementation or routing if already exists.

## Changes
- Extend `--secrets-backend` to support:
  - `k8s:<namespace>/<secretName>` or `k8s:namespace=...`
- Implement secrets-store backend for k8s:
  - key/value entries written as Secret data
  - scope mapping rules documented

## Acceptance Criteria
- `platform install` can write secrets to k8s secrets backend.
- Secrets never appear in logs/output.

## Tests
- Unit tests for mapping + serialization.
- Optional integration tests under k8s harness or mocked client.

## Notes
- Keep backend selection explicit; no auto-detection.
